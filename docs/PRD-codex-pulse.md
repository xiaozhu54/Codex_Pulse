# Codex Pulse 产品需求文档

| 字段 | 内容 |
| --- | --- |
| 状态 | 已确认，可进入实现 |
| 版本 | MVP / 1.0 |
| 日期 | 2026-07-16 |
| 产品形态 | 个人自用的 macOS 菜单栏伴侣应用 |
| 目标应用 | ChatGPT/Codex Desktop，Bundle ID `com.openai.codex` |
| 目标系统 | macOS 14 及以上；首要验收环境为 macOS 26.5.2 |

## 1. Problem Statement

Codex 用户在执行长任务时，无法在不切换窗口、不打开额外页面的情况下持续了解四项直接影响使用决策的信息：weekly 额度余量、模型实时 token 生成速度、当前模型以及当前会话上下文可用比例。

现有信息分散在 Codex 本地会话日志与诊断数据库中，且缺少统一、低干扰的菜单栏呈现。用户需要一个只在 Codex 运行期间出现、默认足够简洁、任务执行时又能提供完整状态的本地工具。该工具必须尊重隐私，不读取凭证、不上传会话内容，也不能通过注入或重签名修改官方应用。

## 2. Goals

1. 在 macOS 菜单栏持续提供可信的 weekly 余量。
2. 在 Codex 有活动任务时，实时显示任务级 token 速度、模型和上下文余量。
3. 支持文字摘要与动态图标两种低干扰显示方式。
4. 通过颜色让用户无需展开面板即可感知 weekly 余量变化。
5. 与 Codex 生命周期联动，并允许用户关闭自动启动监视器。
6. 全程只读本地数据，保证回复内容不会被持久化到插件或发送到网络。

## 3. Non-goals

1. 不修改、注入、替换或重新签名官方 ChatGPT/Codex 应用。
2. 不接管官方 ChatGPT/Codex 状态栏图标，也不保证插件固定排列在 `Help` 菜单之后。
3. 不提供额度通知、声音、闪烁或低余量告警。
4. 不读取 `auth.json`、API Key、Cookie 或其他账号凭证。
5. 不做云同步、远程遥测、使用历史分析或跨设备同步。
6. 不面向公开分发、Mac App Store、公证、自动更新或多用户部署。
7. 不改变模型、任务、额度或 Codex 配置。

## 4. Solution

构建名为 **Codex Pulse** 的原生 macOS 菜单栏伴侣应用。应用通过只读方式观察 `CODEX_HOME` 下的会话 JSONL 和 SQLite 数据，生成统一的状态快照，并根据 Codex 是否有活动任务切换菜单栏内容。

### 4.1 菜单栏状态

#### 动态图标关闭（默认）

- Codex 空闲：显示 `W 99%`。
- Codex 有活动任务：显示 `W 99% · ≈42.3 t/s · GPT‑5.6 · Ctx 89%`。
- 仅 `W 99%` 使用 weekly 渐变色；其余字段使用 macOS 系统文字色。
- 生成中的估算速度带 `≈`；完成校准后移除 `≈`。

#### 动态图标开启

- 菜单栏只显示 Codex Pulse 图标，不显示文字摘要。
- 图标整体使用 weekly 渐变色。
- 悬停图标 300 ms 后显示详情面板。
- 鼠标离开图标及面板 250 ms 后关闭面板。
- 鼠标进入面板时保持显示。
- 单击图标固定打开面板；再次单击或点击外部关闭。
- 右键图标打开设置菜单。

#### 菜单栏限制

Codex Pulse 使用自身的 `NSStatusItem`。排列位置由 macOS 管理，用户可按住 `Command` 拖动。应用不得尝试修改 Codex 的应用菜单或官方状态项。

### 4.2 悬停详情

空闲时显示：

- Weekly 剩余百分比。
- Weekly 重置时间。
- 数据最后更新时间。

任务进行中时额外显示：

- 当前实时或最终 token 速度。
- 最近 5 个已完成模型阶段的平均速度。
- 完整模型名称。
- 上下文可用比例，以及已用 token / 上下文窗口 token。
- 当前会话标题。
- 当前处于“自动跟随”还是“固定会话”模式。
- 当前阶段状态：推理中、生成中、调用工具、等待工具、等待审批或收尾中。

任何字段缺失时仅该字段显示 `—`，不得阻断其他字段更新。

## 5. User Stories

1. 作为 Codex 用户，我希望在菜单栏看到 weekly 剩余比例，以便无需切换窗口就能判断还能使用多少额度。
2. 作为 Codex 用户，我希望在 Codex 空闲时只看到紧凑的 `W 99%`，以便减少菜单栏占用。
3. 作为 Codex 用户，我希望任务开始后自动展开完整摘要，以便立即看到当前任务状态。
4. 作为 Codex 用户，我希望任务结束后摘要自动收回到 `W 99%`，以便界面恢复简洁。
5. 作为 Codex 用户，我希望看到生成中的实时 token 速度，以便判断模型当前输出性能。
6. 作为 Codex 用户，我希望实时速度明确标记为估算值，以免把不完整采样当成精确结果。
7. 作为 Codex 用户，我希望模型阶段完成后速度得到正式 token 数据校准，以便最终结果可信。
8. 作为 Codex 用户，我希望 token 速度涵盖回答、推理和工具调用参数等模型输出，以便反映完整模型工作量。
9. 作为 Codex 用户，我希望工具自身返回的数据不计入 token 速度，以免把外部工具吞吐误认为模型速度。
10. 作为 Codex 用户，我希望工具等待时间不计入生成时长，以免工具延迟拉低模型速度。
11. 作为 Codex 用户，我希望隐藏推理无法实时观测时显示“推理中”，以免插件伪造速度。
12. 作为 Codex 用户，我希望看到当前使用的模型短名称，以便快速确认任务配置。
13. 作为 Codex 用户，我希望在详情中看到完整模型名称，以便处理相似短名称时消除歧义。
14. 作为 Codex 用户，我希望看到当前会话上下文可用比例，以便在接近窗口上限前调整工作方式。
15. 作为 Codex 用户，我希望上下文压缩后比例自动回升，以便数值反映模型下一次请求的实际空间。
16. 作为 Codex 用户，我希望插件自动跟随最近活跃的主会话，以便大多数情况下不需要手动选择。
17. 作为 Codex 用户，我希望自动审查、子代理和其他内部会话不抢占当前会话，以便摘要始终对应我的主任务。
18. 作为同时运行多个任务的用户，我希望最近产生事件的主任务成为当前任务，以便显示最活跃的工作。
19. 作为需要关注特定任务的用户，我希望固定某个会话，以便其他任务活动时显示对象不会变化。
20. 作为固定会话的用户，我希望一键恢复自动跟随，以便回到默认行为。
21. 作为 Codex 用户，我希望任务从 `task_started` 到完成、失败或取消期间均被视为活动，以便状态不提前收起。
22. 作为 Codex 用户，我希望等待工具或审批仍显示为活动任务，以便知道 Codex 尚未结束该任务。
23. 作为 Codex 用户，我希望仅等待下一条用户输入时被视为空闲，以便菜单栏不会长期占用空间。
24. 作为偏好极简菜单栏的用户，我希望开启“动态图标显示”，以便只保留一个图标。
25. 作为动态图标用户，我希望悬停时查看完整信息，以便兼顾简洁与可访问性。
26. 作为动态图标用户，我希望短暂路过图标时不弹出面板，以免误触。
27. 作为需要操作面板的用户，我希望鼠标移入面板后它保持打开，以便选择会话和设置。
28. 作为需要持续查看的用户，我希望单击图标固定面板，以便鼠标移开后面板不消失。
29. 作为 Codex 用户，我希望 weekly 越低时图标越接近橙色，以便通过周边视觉快速感知余量。
30. 作为使用浅色菜单栏的用户，我希望白色图标仍有清晰边界，以便 100% 余量时图标不消失。
31. 作为有色觉差异的用户，我希望余量同时用数字表达，以便不依赖颜色理解状态。
32. 作为 Codex 用户，我希望 Codex 启动时插件界面自动出现，以便无需重复启动两个应用。
33. 作为 Codex 用户，我希望 Codex 退出时插件界面和日志观察自动关闭，以便菜单栏不残留无意义状态。
34. 作为不需要自动联动的用户，我希望关闭“随 Codex 启动”，以便禁用常驻启动监视器。
35. 作为关闭自动联动的用户，我希望当前插件继续运行到 Codex 退出或我手动退出，以免切换开关导致工作中断。
36. 作为关闭自动联动的用户，我希望仍可从 Finder 或 Spotlight 手动启动插件，以便按需使用。
37. 作为隐私敏感用户，我希望插件不读取账号凭证，以便降低本地秘密泄漏风险。
38. 作为隐私敏感用户，我希望流式文本只在内存中分词且立即丢弃，以便插件不形成第二份会话记录。
39. 作为隐私敏感用户，我希望插件没有网络上传行为，以便所有数据都停留在本机。
40. 作为使用自定义 `CODEX_HOME` 的用户，我希望在设置中选择数据目录，以便插件能读取非默认安装。
41. 作为遇到日志轮转的用户，我希望插件自动重新发现当前日志文件，以便无需重启。
42. 作为遇到局部数据缺失的用户，我希望不可用字段显示 `—`，以便明确区分未知与零。
43. 作为遇到 Codex 更新的用户，我希望不兼容的数据源被隔离降级，以便一个解析器失败不会导致应用崩溃。
44. 作为用户，我希望数据恢复后界面自动恢复，以便不需要手动刷新或重启。
45. 作为个人开发者，我希望插件使用原创 `W + Codex 几何交织风格` 图标，以便与官方状态项区分。

## 6. Functional Requirements

### FR-1 Weekly 余量

1. 从 Codex `token_count.rate_limits` 中选择 `window_minutes == 10080` 的七天限额。
2. 若存在多个候选，优先选择 `limit_id == "codex"` 且时间戳最新的记录。
3. 计算 `remaining_percent = clamp(100 - used_percent, 0, 100)`。
4. 菜单栏四舍五入到整数；详情允许显示 1 位小数。
5. 显示 `resets_at` 对应的本地日期时间。
6. 不得用小时窗口或其他额度冒充 weekly；找不到七天窗口时显示 `—`。

### FR-2 Token 速度

1. 监听带纳秒时间戳的 Responses 流式事件。
2. 实时采样周期为 500 ms，滑动窗口为最近 2 秒。
3. 实时 token 统计覆盖所有可观测模型输出事件，包括：
   - `response.output_text.delta`
   - `response.reasoning_summary_text.delta`
   - `response.function_call_arguments.delta`
   - `response.custom_tool_call_input.delta`
4. 工具返回内容和工具执行时间不计入。
5. 流式内容只进入内存分词器；不得写入插件日志、缓存、数据库或诊断文件。
6. 生成中的值显示为 `≈x.x t/s`。
7. 当前阶段没有可观测 token 且处于隐藏推理时显示 `推理中…`。
8. 模型阶段完成后，使用 Codex 正式 `output_tokens`（包含 `reasoning_output_tokens`）和该模型响应阶段的有效持续时间计算最终平均速度。
9. 有效持续时间以单次模型响应的 `created/in_progress` 至 `completed` 为边界；多个模型响应分别计速，工具间隔不合并进持续时间。
10. 保留最近 5 个已完成模型阶段的速度，仅用于内存中的移动平均；应用退出后不保存。
11. 无法获得流式诊断事件时，实时值显示 `—`，阶段结束后若正式统计可用则仍显示最终速度。

### FR-3 模型

1. 从当前会话最新 `turn_context.model` 或线程元数据读取模型。
2. 菜单栏使用可读短名称，例如 `gpt-5.6-sol` 显示为 `GPT‑5.6`；不得把未知模型错误归并。
3. 详情面板显示日志中的完整模型标识。
4. 模型在任务中发生切换时，于下一条模型事件后更新。

### FR-4 上下文比例

1. 使用当前会话最新一条完整 token 统计。
2. 计算：

   `available_percent = clamp(1 - (last_input_tokens + last_output_tokens) / model_context_window, 0, 1) × 100`

3. 菜单栏显示取整百分比；详情显示 1 位小数及 token 绝对值。
4. 上下文压缩后按最新 token 统计重新计算，不保持单调下降。
5. 没有完成过请求或缺少窗口大小时显示 `—`。

### FR-5 当前会话与活动任务

1. 默认模式为自动跟随。
2. 候选会话必须满足：非归档、用户可见、主会话、存在用户事件。
3. 排除自动审查、子代理、内部 agent job 及其他内部线程。
4. 自动跟随选择最近产生可识别任务事件的候选会话。
5. 活动任务从 `task_started` 开始，到对应的 `task_complete`、失败或取消事件结束。
6. 思考、输出、工具调用、等待工具和等待用户审批均属于活动任务。
7. 回答完成且只等待新的用户输入时属于空闲。
8. 用户可从详情面板固定一个候选会话。
9. 固定模式只以被固定会话为准，即使其他会话更活跃。
10. 被固定会话删除或无法读取时，显示不可用状态并提供“恢复自动跟随”，不得静默切换。

### FR-6 动态图标与颜色

1. 设置项名称必须为 `动态图标显示`，默认关闭并持久化。
2. 图标使用项目资产 `assets/weekly-codex-icon.png` 作为视觉母版。
3. 颜色采用 12 个色标：2 个端点和 10 个中间色。

| Weekly 剩余 | 色值 |
| ---: | --- |
| 100% | `#FFFFFF` |
| 90.9% | `#FFF9F2` |
| 81.8% | `#FFF1E3` |
| 72.7% | `#FFE7D0` |
| 63.6% | `#FFDABD` |
| 54.5% | `#FFCCA8` |
| 45.5% | `#FFBD91` |
| 36.4% | `#FFAD78` |
| 27.3% | `#FF9D5F` |
| 18.2% | `#FF8D45` |
| 9.1% | `#FF812E` |
| 0% | `#FF7417` |

4. 精确余量在相邻色标间使用 OKLab/OKLCH 感知色彩空间连续插值。
5. 100% 必须为白色，0% 必须为橙色。
6. 通过细描边或等效的自适应高对比处理保证白色在浅色菜单栏上可见。
7. 菜单栏处于选中/按下状态时使用系统高对比选中色。
8. 颜色不得成为唯一信息载体；可展开状态必须同时提供百分比。

### FR-7 生命周期

1. 使用 Bundle ID `com.openai.codex` 识别目标应用，不依赖显示名“ChatGPT”。
2. 设置项名称为 `随 Codex 启动`，默认开启。
3. 开启时注册登录启动监视器；监视器通过系统应用生命周期通知监听 Codex，不使用高频进程轮询。
4. Codex 已运行时，插件显示状态栏项目并启动数据观察。
5. Codex 退出时，插件移除状态栏项目、停止数据观察并释放会话内存；轻量启动监视器保留。
6. 关闭 `随 Codex 启动` 时立即注销登录启动监视器。
7. 关闭开关不强制终止当前插件；插件继续到 Codex 退出、用户选择“退出插件”或手动关闭。
8. 关闭开关后允许用户从 Finder 或 Spotlight 手动启动插件。

### FR-8 设置

设置菜单至少包含：

1. `动态图标显示` 开关。
2. `随 Codex 启动` 开关。
3. 当前 `CODEX_HOME` 路径及“选择文件夹”。
4. 会话模式：自动跟随 / 当前固定会话。
5. “恢复自动跟随”。
6. “退出插件”。

设置只持久化开关、`CODEX_HOME`、固定会话 ID 和界面偏好。

## 7. Implementation Decisions

### 7.1 技术栈

- 使用 Swift、SwiftUI 和必要的 AppKit API 构建原生菜单栏应用。
- 应用以 accessory/UIElement 形态运行，不显示 Dock 图标和普通主窗口。
- 使用 `NSStatusItem` 实现菜单栏内容，使用 `NSPopover` 或无标题浮层实现可交互悬停详情。
- 使用 `NSWorkspace` 应用生命周期通知识别 Codex 启停。
- 使用系统登录项服务注册轻量启动监视器。

### 7.2 模块边界

1. **Codex Lifecycle Monitor**：只负责目标 Bundle ID 的启停与监视器开关。
2. **Codex Data Source Adapters**：只读发现并解析 JSONL、状态数据库和诊断数据库；对文件名和 schema 版本差异做适配。
3. **Session Resolver**：过滤内部会话、判断活动任务、执行自动跟随和固定会话规则。
4. **Metrics Engine**：计算 weekly、token 速度、模型和上下文比例，不接触 UI。
5. **Status Snapshot Store**：把多个数据源归一为单一、不可变的用户可见状态快照。
6. **Menu Bar Presenter**：只根据快照和用户偏好渲染文字、图标、悬停面板及设置。
7. **Preferences Store**：只保存允许的偏好字段。

UI 不直接查询 SQLite 或读取 JSONL；所有展示必须来自统一状态快照。

### 7.3 数据源

- 默认根目录为 `~/.codex`，允许选择其他 `CODEX_HOME`。
- 会话事件来自 `sessions/**/*.jsonl`，包括任务、模型和 token 统计。
- 会话索引及元数据来自兼容的 `state_*.sqlite`。
- 实时 SSE 事件来自兼容的 `logs_*.sqlite`。
- SQLite 必须只读打开，兼容 WAL，设置短 `busy_timeout`，不得阻塞 Codex 写入。
- 使用文件系统事件和增量游标观察更新，不重复扫描完整历史。
- 日志轮转或 inode 变化时重新发现数据源，并从安全边界恢复。
- 解析器必须按能力检测字段，不得只依赖当前具体版本号。

### 7.4 隐私与安全

- 明确拒绝打开 `auth.json` 及任何已知凭证文件。
- 应用不实现网络上传、遥测和云 API。
- 不记录原始会话文本、流式 delta、工具参数或工具结果。
- 分词输入只在进程内存中存在，完成统计后立即释放。
- 应用日志只允许记录事件类型、时间、错误码和不可逆的结构信息，不得包含用户内容。
- 数据库和会话文件只读；不得执行迁移、修复或写回。
- 个人自用版本不以公开分发为目标；若未来启用 App Sandbox，需以用户授权目录和安全作用域书签替代当前本地读取方式。

### 7.5 兼容与降级

- 每项指标独立维护可用性和时间戳。
- 数据源不可用、schema 不兼容或字段缺失时，该指标显示 `—`。
- 不得把未知值显示为 0。
- 解析失败不得导致应用崩溃或影响 Codex。
- 数据恢复后自动重建观察并刷新 UI。
- 插件不得依赖 `ps`、屏幕抓取、辅助功能注入或 UI 自动化判断 Codex 状态。

## 8. State Model

| Codex 状态 | 任务状态 | 动态图标 | 菜单栏 | 悬停/点击内容 |
| --- | --- | --- | --- | --- |
| 未运行 | 任意 | 任意 | 不显示插件状态项 | 无 |
| 已运行 | 空闲 | 关 | `W 99%` | Weekly、重置时间、更新时间 |
| 已运行 | 活动 | 关 | 完整摘要 | 完整任务详情 |
| 已运行 | 空闲 | 开 | 着色图标 | Weekly、重置时间、更新时间 |
| 已运行 | 活动 | 开 | 着色图标 | 完整任务详情 |
| 已运行 | 活动且隐藏推理 | 关 | 速度字段显示 `推理中…` | 显示阶段状态 |
| 已运行 | 数据部分缺失 | 任意 | 对应字段 `—` | 显示数据源状态和更新时间 |

## 9. Non-functional Requirements

### 性能

- Codex 空闲且状态稳定时，插件平均 CPU 使用率目标低于 1%。
- 启动监视器必须事件驱动，不得每秒轮询进程列表。
- 从日志写入可观测事件到菜单栏更新的 P95 延迟不超过 750 ms。
- Codex 启动后 2 秒内显示插件状态项；Codex 退出后 2 秒内移除。
- 增量观察不得重复解析完整会话历史。

### 稳定性

- Codex 数据库繁忙、WAL 切换、JSONL 半行写入或文件轮转不得导致崩溃。
- 读取器必须等待完整 JSON 行后再解析。
- 当前会话解析失败时不得自动选择内部线程作为替代。
- 插件异常不得影响 Codex 进程或文件。

### 可访问性

- 菜单栏和面板信息必须有 VoiceOver 标签。
- 颜色与百分比并用。
- 面板支持键盘操作和 Escape 关闭。
- 遵守 Reduce Motion；MVP 不使用持续动画。

## 10. Testing Decisions

### 10.1 主要测试缝

采用一个最高层级的行为测试缝：向完整应用输入临时 `CODEX_HOME` 测试夹具和模拟的 Codex 生命周期事件，观察最终 `StatusSnapshot` 与菜单栏/悬停面板的用户可见状态。

该测试缝覆盖：

- 数据源发现和增量读取。
- weekly、速度、模型和上下文计算。
- 主会话过滤、自动跟随与固定会话。
- 空闲/活动状态切换。
- 动态图标开关与摘要格式。
- 部分数据缺失、日志轮转和 schema 不兼容降级。
- Codex 启停及启动监视器开关。

测试应断言外部行为和状态快照，不断言私有类、SQL 语句或内部调用顺序。

### 10.2 测试夹具

至少提供以下脱敏、最小化夹具：

1. 空闲会话，weekly 数据可用。
2. 活动会话，持续写入可观测流式 delta。
3. 含隐藏 reasoning tokens 的已完成阶段。
4. 工具调用与长工具等待。
5. 上下文压缩前后两组 token 统计。
6. 两个并发主会话与一个自动审查线程。
7. 固定会话失效。
8. SQLite WAL 更新、busy 和轮转。
9. JSONL 半行后补全。
10. weekly 缺失但其他指标可用。
11. 日志 schema 缺字段。

夹具不得包含真实用户提示、回答、路径、账号或凭证。

### 10.3 计算测试

- 验证 weekly 剩余、边界 clamp 和重置时间本地化。
- 验证 500 ms 采样与 2 秒滑动窗口。
- 验证工具等待时间未进入最终生成时长。
- 验证正式 output tokens 对实时估算的校准。
- 验证上下文公式、四舍五入及压缩后回升。
- 验证 12 色标端点和中间插值，特别是 100% 白色与 0% 橙色。

### 10.4 macOS UI 冒烟测试

在真实 macOS 环境验证：

1. 状态项在 Codex 启停时出现/消失。
2. 悬停延迟和自动关闭时间。
3. 点击固定面板、外部点击关闭、右键设置。
4. Command 拖动状态项。
5. 浅色、深色及选中菜单栏下的图标可见性。
6. VoiceOver 和键盘操作。
7. 登录监视器开启、关闭及手动启动路径。

## 11. Acceptance Criteria

1. Codex 未运行时不存在 Codex Pulse 状态项；Codex 启动后 2 秒内出现。
2. Codex 空闲时，在动态图标关闭模式显示 `W <剩余百分比>`，不显示另外三项。
3. 主任务开始后 750 ms 内切换为完整摘要；任务结束后恢复空闲摘要。
4. 动态图标开启后菜单栏文字完全隐藏，只保留原创图标。
5. 悬停 300 ms 打开面板，离开图标及面板 250 ms 后关闭；点击可固定。
6. Weekly 为 100% 时颜色为 `#FFFFFF`，为 0% 时颜色为 `#FF7417`，中间连续变化。
7. 生成过程中速度至少每 500 ms 有机会更新，并显示 `≈`。
8. 阶段完成后使用正式全部 output tokens 校准最终速度。
9. 工具返回及工具等待不进入模型速度统计。
10. 模型、上下文和当前会话均来自被选中的主会话。
11. 自动审查与子代理活动不会抢占主会话。
12. 固定会话后其他会话活动不会改变显示对象。
13. 插件读取失败时只降级对应字段，不崩溃、不写回、不影响 Codex。
14. 插件不会打开凭证文件，不会上传或持久化会话内容。
15. 关闭 `随 Codex 启动` 后登录监视器被注销，之后 Codex 启动不会自动显示插件。

## 12. Risks and Mitigations

| 风险 | 影响 | 缓解措施 |
| --- | --- | --- |
| Codex 本地日志属于内部接口，升级可能变更 | 指标暂时不可用 | 数据源适配层、能力检测、独立降级、脱敏回归夹具 |
| 诊断日志可能被禁用或轮转 | 实时速度不可用 | 实时显示 `—`；阶段完成后用正式统计恢复最终速度 |
| 隐藏推理无法逐 token 实时观察 | 实时速度不完整 | 显示“推理中”；阶段完成后用正式 reasoning/output tokens 校准 |
| 多任务并发造成当前任务歧义 | 显示错误会话 | 主会话过滤、最近事件规则、手动固定 |
| 白色图标在浅色背景不可见 | 丧失状态提示 | 自适应描边和选中态高对比 |
| 读取活跃 SQLite 影响 Codex | 性能或锁竞争 | 只读连接、短查询、busy timeout、增量游标、不得迁移 |

MVP 完成意味着：在目标 Mac 上可手动安装和运行；所有 Acceptance Criteria 通过；主要行为测试缝及 macOS UI 冒烟测试通过；隐私检查确认无凭证访问、无会话内容持久化、无网络上传；PRD 中的 Out of Scope 未被隐式引入。
