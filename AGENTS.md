# AGENTS.md

## Mission

在本仓库实现个人自用的 macOS 菜单栏伴侣应用 **Codex Pulse**。开始任何设计、实现或评审前，必须完整阅读 `docs/PRD-codex-pulse.md`。PRD 是产品行为与验收标准的唯一事实来源；不要擅自扩展范围。

## Non-negotiable Boundaries

- 不修改、注入、替换或重新签名 `/Applications/ChatGPT.app`。
- 只通过 Bundle ID `com.openai.codex` 识别目标应用。
- Codex 数据源必须只读；绝不迁移、修复、截断或写回 Codex 文件。
- 禁止读取 `auth.json`、API Key、Cookie、令牌或其他凭证。
- 禁止上传遥测、日志、会话内容或指标。
- 禁止持久化提示、回答、流式 delta、工具参数和工具结果。
- 原始内容只可在内存中用于即时 token 统计，统计后立即释放。
- 不通过 `ps` 轮询、屏幕抓取、辅助功能注入或 UI 自动化判断 Codex 生命周期。
- 不实现 PRD 明确排除的通知、云同步、公开分发或官方图标接管。

## Product Invariants

- Codex 未运行时不显示插件状态项。
- 空闲且动态图标关闭时只显示 `W <weekly 剩余百分比>`。
- 活动任务且动态图标关闭时显示 `W 99% · ≈42.3 t/s · GPT‑5.6 · Ctx 89%` 结构。
- 动态图标开启时只显示着色图标，详细信息通过悬停/点击面板提供。
- `动态图标显示` 默认关闭；`随 Codex 启动` 默认开启。
- 颜色端点固定为 100% `#FFFFFF`、0% `#FF7417`，并使用 PRD 中 10 个中间色标。
- 未知值显示 `—`，不得显示为 0 或用其他窗口数据冒充 weekly。
- 自动跟随只选择用户可见的主会话；内部审查、子代理和 agent job 不得抢占。
- 固定会话优先级高于自动跟随，失效时不得静默切换。

## Architecture

保持以下单向依赖：

`只读数据源 → 会话解析/指标计算 → StatusSnapshot → 菜单栏展示`

- UI 不得直接访问 SQLite 或 JSONL。
- 数据源适配器应按能力和字段检测兼容 Codex 版本，不要把 `state_5.sqlite` 或 `logs_2.sqlite` 当成永久文件名。
- SQLite 使用只读、WAL 兼容的短查询；避免长事务和全表重复扫描。
- JSONL 只解析完整行，支持半行写入、轮转和 inode 变化。
- 以不可变的统一 `StatusSnapshot` 作为主要产品测试缝。
- 启动监视器应事件驱动，并与菜单栏 UI/数据观察生命周期分离。

## Technology Direction

- 首选 Swift、SwiftUI 与必要的 AppKit API。
- 使用 `NSStatusItem` 呈现菜单栏，使用 `NSPopover` 或等效无标题浮层呈现详情。
- 使用 `NSWorkspace` 生命周期通知监听 `com.openai.codex`。
- 使用系统登录项机制实现可关闭的轻量启动监视器。
- 使用 macOS 系统颜色、辅助功能标签和键盘交互；遵守 Reduce Motion。
- 运行时图标资源使用 `assets/weekly-codex-icon.png`，不得使用色键源文件。

## Testing Rules

- 优先从最高层级测试用户可见行为，不测试私有实现细节。
- 主要测试缝：向完整应用输入临时 `CODEX_HOME` 夹具和模拟生命周期事件，断言 `StatusSnapshot` 及最终菜单栏/面板状态。
- 测试夹具必须完全脱敏，不得从真实 `~/.codex` 复制用户内容或凭证。
- 必须覆盖并发主会话、内部会话过滤、固定会话、日志轮转、SQLite WAL/busy、JSONL 半行、上下文压缩、隐藏推理校准和字段独立降级。
- 颜色测试必须验证两个端点、全部色标和连续插值。
- 对悬停、点击、右键、Command 拖动、登录监视器和浅/深色菜单栏执行真实 macOS 冒烟测试。
- 修改实现后运行与改动最接近的测试；交付前运行完整测试套件。

## Privacy Review Checklist

每个涉及数据读取、日志或错误上报的改动都必须回答：

1. 是否可能打开或间接复制凭证文件？
2. 是否可能把用户内容写入插件日志、测试失败输出、缓存或崩溃报告？
3. 是否引入网络请求、遥测 SDK 或远程依赖？
4. 是否可能对 Codex 数据库或会话文件加写锁？
5. 数据源不兼容时是否安全降级，而不是尝试修复 Codex 文件？

任一答案存在风险时，不得合并，除非 PRD 已更新且用户明确确认新的边界。

## Working Practices

- 保持改动小而可验证，不顺手实现 Out of Scope 功能。
- 代码中的产品术语使用 PRD 定义：weekly、活动任务、主会话、自动跟随、固定会话、动态图标、状态快照。
- 新增依赖前说明必要性；优先使用系统框架，避免为简单能力引入大型包。
- 不把当前本机绝对路径写入产品代码。
- 不把当前 Codex schema 版本当作公开稳定 API；任何解析假设都必须由脱敏夹具覆盖。
- 如果实现发现 PRD 与实际 Codex 数据源冲突，先记录证据并请求产品决策，不要静默改变指标定义。
